# Flow-Py 서비스 화면 가이드 (UI 디자인 참고용)

이 문서는 UI 개선(디자인 파일 제작)을 위해 현재 구현된 모든 화면의 목적·구성·데이터·인터랙션을
정리한 것이다. 코드 기준(2026-07-16) 실제 동작을 기술한 것이며, 화면에 없는 기능은 포함하지 않았다.

---

## 0. 서비스 개요

**Flow-Py**는 순서도(플로우차트) 기반으로 파이썬을 가르치는 학원용 웹 서비스다.
- 선생님은 문제(순서도/파이썬 코드/블럭 코딩)를 만들어 반에 배정하고, 학생 제출을 채점·관리한다.
- 학생은 배정된 문제를 웹 브라우저에서 바로 풀고(Pyodide로 파이썬을 브라우저에서 실행), 제출하면
  자동 채점되어 포인트를 받는다. 포인트로 상점에서 상품을 구매 요청할 수 있다.
- 역할은 `teacher` / `student` 둘뿐이며, 로그인 시 역할에 따라 완전히 다른 메뉴 구성으로 진입한다.
- 모바일에서는 데스크톱 사이드바 대신 하단 탭바로 전환되는 반응형 레이아웃이다(2026-07-16 추가).
- 공지사항/쪽지/구매요청·승인/수업 30분 전 알림은 웹 푸시(Web Push)로 발송된다.

## 1. 디자인 시스템 기본값

- **컴포넌트 베이스**: shadcn/ui 스타일(Radix 기반) + Tailwind CSS. `Button`, `Input`, `Label`,
  `Dialog`, `Card`, `Textarea`, `Tabs` 등 재사용 프리미티브를 씀.
- **주 색상(Primary)**: 오렌지 계열 `hsl(16 100% 50%)` (라이트) / `hsl(16 100% 55%)` (다크).
  버튼, 활성 메뉴 항목, 선택 상태, 포인트/강조 텍스트에 사용.
- **카드 스타일**: `rounded-xl border bg-card p-4 shadow-sm` — 대시보드 위젯, 상점 카드, 반/문제
  목록 항목 등 대부분의 콘텐츠 블록이 이 톤(둥근 모서리 + 얇은 테두리 + 옅은 그림자)을 공유.
- **그리드 카드 뷰(상점)**: `grid-cols-2 sm:grid-cols-3 md:grid-cols-4 lg:grid-cols-5` — 화면
  폭에 따라 2~5열. 정사각 썸네일(`aspect-square`) + 이름 + 가격/재고 + 액션 버튼.
- **다이얼로그**: 등록/수정/할당류 액션은 대부분 페이지 이동 없이 모달(`Dialog`)로 처리(상세
  페이지 없음 원칙 — 리스트/카드에서 바로 액션).
- **토스트**: `sonner` 사용, 성공/실패 피드백은 화면 상단 중앙에 짧게 표시. 별도 알림 배너 없음.
- **아이콘**: `lucide-react` 라인 아이콘, 크기는 보통 `size-4`(16px) / `size-3.5`(14px, 인라인
  버튼) / `size-5`(20px, 하단 탭바).
- **반응형 브레이크포인트**: `md`(768px) 기준으로 데스크톱 사이드바 ↔ 모바일 하단 탭바 전환.

## 2. 사용자 역할 & 내비게이션 구조

### 선생님 사이드바 메뉴 (데스크톱)
1. 대시보드 (홈, `/dashboard`)
2. 반 관리 (`/classes`)
3. 문제 관리 (`/problems`)
4. 타자 연습 — *준비 중(비활성, 클릭 시 토스트만)*
5. 포인트 상점 (`/shop`)
6. 알림 (`/notifications`)

모바일 하단 탭: 위 6개 전부 그대로 노출(선생님은 축소 안 함).

### 학생 사이드바 메뉴 (데스크톱)
1. 대시보드 (홈, `/student`)
2. 내 수업 (`/myclass`)
3. 순서도 연습 (`/practice/flowchart`)
4. 파이썬 코딩 (`/practice/general`)
5. 블럭 코딩 (`/practice/block`)
6. 타자 연습 — *준비 중*
7. 포인트 상점 (`/student/shop`)
8. 알림 (`/student/notifications`)

모바일 하단 탭(학생): **대시보드, 내 수업, 포인트 상점 3개만** — 화면이 좁아 핵심만 남기고 나머지는
데스크톱 사이드바에서만 접근 가능하도록 축소(2026-07-16 결정).

## 3. 공통 셸(Shell) 컴포넌트

### 3.1 AppShell (`src/components/layout/AppShell.tsx`)
모든 로그인 후 화면(문제 풀이 화면 `Solve`, 라이브 수업 화면 `LiveClass` 제외)을 감싸는 뼈대.

- **데스크톱(md 이상)**
  - 좌측 고정 사이드바: 로고(Flow-Py) + 접기/펼치기 토글(`PanelLeftClose/Open`), 너비 `w-56`
    ↔ 접으면 `w-16`(아이콘만). 대시보드 항목 + 구분선 + 역할별 메뉴 목록.
  - 메뉴 항목은 아이콘 + 라벨, 활성 항목은 배경이 primary 색으로 채워짐. "준비 중" 항목은 라벨
    옆에 회색 작은 텍스트 배지.
  - 우측 상단 `Header`(아바타 드롭다운), 본문은 스크롤 가능한 `main` 영역.
- **모바일(md 미만)**
  - 사이드바 숨김, 화면 하단에 고정 탭바(`fixed inset-x-0 bottom-0`) 표시. 아이콘(20px) + 아주
    작은 라벨(10px), 활성 탭은 primary 색 텍스트. 항목 수가 많아도 넘치면 가로 스크롤.
  - 본문 하단에 탭바 높이만큼 여백(`pb-16`) 추가해 콘텐츠 가림 방지.

### 3.2 Header (`src/components/layout/Header.tsx`)
모든 로그인 화면 상단 바(높이 56px). 좌측에 선택적 타이틀, 우측에 아바타 원(이니셜 + 사용자
지정 색상) + 이름 + 드롭다운 화살표. 클릭 시 드롭다운:
- 아바타 색상 선택(팔레트 스와치), 표시 이름 인라인 편집(연필 아이콘 → 입력창 → 체크로 저장).
- 로그아웃 버튼(빨간 텍스트).
- 사용자 설정은 로컬 저장(`profile-prefs`)이라 기기별로 다를 수 있음.

## 4. 인증 화면

### 4.1 랜딩 (`src/pages/Landing.tsx`, `/`)
- 비로그인 시에만 보이는 마케팅 성격 첫 화면. 상단 헤더(로고 + 로그인/회원가입 버튼),
  중앙 히어로 타이틀 "순서도로 배우는 파이썬" + 부제, 3개 특징 카드(드래그드롭 순서도 / 웹 IDE +
  실행 / 자동 채점), 하단 푸터.
- 로그인 상태면 이 화면 대신 역할별 홈으로 즉시 리다이렉트.

### 4.2 로그인/회원가입 드롭다운 (`AuthDropdown` + `AuthForm`)
- 헤더의 "로그인"/"회원가입" 버튼 클릭 시 우측 상단에 작은 팝오버 폼(페이지 이동 없음).
- 탭 전환(로그인/회원가입), 아이디(이메일 아님, 내부적으로 `아이디@flowpy.local`로 매핑)/비밀번호
  입력. 회원가입 시 "선생님으로 가입" 체크 + 선생 가입 코드 입력 필드가 추가로 나타남.
- 성공 시 팝오버 닫고 홈으로 이동(역할별 리다이렉트는 라우트 가드가 처리).

## 5. 선생님 화면

### 5.1 대시보드 (`src/pages/Dashboard.tsx`, `/dashboard`)
4열 반응형 벤토 그리드(위젯 카드 모음). 위젯 목록과 각각의 내용:

| 위젯 | 크기 | 내용 |
|---|---|---|
| 수업 시간표 | 2×2 | `ScheduleCalendar` — 아래 5.1.1 참조 |
| 접속 중인 학생 | 2×1 | 실시간 온라인 학생 이름 칩(초록 점 + 이름), presence 기반 |
| 내 문제 (통계) | 1×1 | 총 문제 수 + "발행 N개" |
| 등록 학생 (통계) | 1×1 | 전체 학생 수 |
| 최근 제출 현황 | 2×2 | 최근 제출 8건: 학생명 · 문제명 · 통과 여부 아이콘 · 점수 |
| 빠른 실행 | 2×1 | "문제 만들기" / "문제 관리" / "반 관리" 바로가기 버튼 |
| 공지사항 | 2×1 | `AnnouncementsPanel`(쓰기 가능 모드) |
| 학사 일정 | 2×1 | `AcademicEventsPanel`(쓰기 가능 모드) |
| 쪽지함 | 2×1 | `MessageCenter`(수신 대상: 전체 학생) |

각 위젯은 공통 `Bento` 래퍼(아이콘 + 제목 헤더 + 카드 배경)로 통일된 톤.

#### 5.1.1 수업 시간표 (`ScheduleCalendar`)
- 로컬(브라우저별) 저장되는 개인용 캘린더. **1주일 뷰**(요일 7칸, 오늘 강조)와
  **30일(월) 뷰** 전환 가능, 좌우 화살표로 주/월 이동, "오늘" 버튼.
- 빈 날짜 칸 클릭 → 일정 추가 모달(제목, 날짜, 시간, 색상 6종 중 선택). 기존 일정 칩 클릭 →
  같은 모달로 수정/삭제.
- 주의: 이 캘린더는 "반 관리"의 수업 요일/시간(알림용 스케줄)과는 별개의 개인 메모용 캘린더.

### 5.2 반 관리 (`src/components/admin/ClassManager.tsx`, `/classes`)
좌우 2단 레이아웃(상세 페이지 없이 한 화면에서 전부 처리):

- **좌측 패널(고정 폭)**: 반 목록. "새 반" 버튼(누르면 즉시 "새 반 N" 생성 후 선택). 각 행은
  아이콘 + 이름(인라인 편집 가능, 연필 아이콘) + 삭제(휴지통) — 삭제 시 확인창.
- **우측 패널**: 선택한 반의 상세.
  - **수업 시간** 설정 바: 요일 드롭다운 + 시간 입력(`<input type="time">`) — 설정하면 수업 시작
    30분 전 학생에게 웹 푸시 알림이 발송됨(안내 문구 동반).
  - **등록 학생** 섹션: 이름 칩 목록(각 칩에 포인트 부여 아이콘 + 등록 해제 아이콘), "학생 등록"
    버튼 → 학생 검색 + 체크박스 다중 선택 모달.
  - **할당된 문제** 섹션: 제목 + 발행 상태(발행됨/미발행) 목록, "문제 할당" 버튼 → 폴더별로 그룹
    핑된 문제 체크박스 모달.
  - **수업하기** 버튼: 라이브 모니터링 화면(5.3)으로 이동.
  - 포인트 부여 모달: 학생 이름 표시, 포인트(양/음수 가능해 보임) + 사유 입력.

### 5.3 수업하기 — 라이브 모니터링 (`src/pages/LiveClass.tsx`, `/classes/:classId/live`)
전체 화면(사이드바 없음, 전용 헤더만). 목적: 실시간으로 학생 화면을 지켜보기.

- **좌측 고정 목록**: 반에 등록된 학생, 온라인이면 초록 점 표시. 온라인 학생만 클릭 가능,
  최대 4명까지 동시 관전 선택(토글 방식).
- **우측 그리드**: 선택 인원 수에 따라 1열/2열/2×2 자동 배치. 각 타일은:
  - 상단 바: 학생 이름 + 현재 풀고 있는 문제명 + 문제 패널 펼침/접기 토글 + 닫기(✕).
  - 문제 패널(펼쳤을 때): 순서도 문제면 읽기 전용 순서도 캔버스, 일반 문제면 설명 텍스트.
  - 하단: 학생 코드 에디터를 **읽기 전용**으로 실시간 반영(Supabase Realtime 브로드캐스트).
  - 아직 문제를 안 풀고 있으면 "아직 문제를 풀고 있지 않습니다" 플레이스홀더.

### 5.4 문제 관리
**현재 실제로 메뉴에 연결된 화면**은 `ProblemManager`(폴더 트리 버전, `/problems`)이다.

#### 5.4.1 `ProblemManager` (`src/components/admin/ProblemManager.tsx`, `/problems`)
3단 레이아웃:
- **좌측(폴더 트리, 고정 폭)**: "전체" + 대분류 폴더(순서도/파이썬/블럭, 자동 생성)와 하위 폴더
  트리. 폴더 펼침/접기(화살표), 하위 폴더 추가(아이콘, prompt 입력), 폴더 삭제(기본 대분류
  제외). 문제를 드래그해서 폴더에 드롭하면 해당 폴더로 이동 + 카테고리 자동 변경.
- **가운데(문제 목록, 고정 폭)**: 선택된 폴더의 문제만 필터링. 각 행은 발행 여부 점(초록/회색) +
  제목 + 발행 전환 토글(지구본/눈-감김 아이콘) + 삭제. 드래그 가능(폴더 이동용).
- **우측(에디터, 나머지 전체)**: 아래 5.4.3 `ProblemEditor` 그대로 재사용.

#### 5.4.2 `TeacherProblems` (`/teacher`) — *레거시, 현재 메뉴에 링크 없음*
`ProblemPanel`(폴더 없는 단순 목록) + `ProblemEditor` 조합의 더 단순한 옛 버전. 코드는 남아
있으나 사이드바 메뉴 어디에서도 연결되지 않음 → **디자인 갱신 대상에서 제외 권장**, 필요 시 삭제
또는 통합 검토.

#### 5.4.3 문제 에디터 (`src/components/teacher/ProblemEditor.tsx`)
상단 툴바: 제목 입력, 카테고리 선택(순서도/파이썬/블럭), 만점 포인트 숫자 입력, 우측에
"제출 현황"(다이얼로그, 5.4.4) / "발행·발행취소" / "저장" 버튼.

본문은 카테고리에 따라 레이아웃이 다름:
- **순서도 카테고리**: 좌측 = 순서도 편집 캔버스(7장 참조, 편집 가능), 우측 = 탭 3개
  ("코드·실행" = 시작 코드 + Pyodide 실행 콘솔, "문제 설명" = 학생에게 보일 설명 텍스트,
  "채점 (N)" = 테스트케이스 에디터).
- **파이썬/블럭 카테고리**: 좌측 = 설명 텍스트 + 채점 테스트 에디터(세로 배치), 우측 = 시작 코드
  에디터 + 실행 콘솔.

#### 5.4.4 제출 현황 다이얼로그 (`TeacherSubmissions`)
문제 에디터에서 "제출 현황" 클릭 시 모달. 학생별 제출 목록(이름, 제출 시각, 통과/총 테스트,
점수) → 행 클릭 시 펼쳐져서 테스트케이스별 통과 여부 + 제출된 코드 원문(읽기 전용) 표시.

#### 5.4.5 채점 테스트 에디터 (`GradingTestsEditor`)
테스트케이스 카드 목록: 이름, 배점, stdin 입력, 기대 stdout 텍스트영역. "테스트 추가"/삭제.
안내 문구: 테스트가 없으면 제출 점수는 0점.

### 5.5 포인트 상점 — 선생님 (`src/components/admin/ShopManager.tsx`, `/shop`)
상세 페이지 없이 카드 뷰에서 등록/수정/삭제/구매수락까지 전부 처리(사용자 명시 요구사항).

- 상단: 제목 + "상품 등록" 버튼.
- **상품 카드 그리드**(2~5열): 정사각 썸네일 + 이름 + "N P · 재고 N" + 인라인 "수정"/삭제 버튼.
- **등록/수정 모달**: 이름, 이미지(파일 업로드 → Supabase Storage 버킷에 저장 후 미리보기),
  포인트, 수량. 업로드 중엔 저장 버튼 비활성.
- **구매 요청 섹션**(카드 그리드 아래): 대기 중인 요청만 리스트로 표시(학생명 · 상품명 + 수락/거절
  버튼). 수락 시 DB 트리거가 재고 차감 + 포인트 차감을 원자적으로 처리.

### 5.6 알림 설정 (`src/components/notifications/NotificationSettings.tsx`, `/notifications`,
학생 쪽은 `/student/notifications`에 동일 컴포넌트 재사용)
단일 카드: 종 아이콘(켜짐: 채워진 primary색 종 / 꺼짐: 슬래시 종) + "브라우저 알림 받기" 제목 +
설명("공지사항, 쪽지, 구매 요청/승인, 수업 시작 30분 전 알림을 이 기기로 받습니다.") + 토글
버튼("알림 켜기"/"알림 끄기"). 브라우저가 웹 푸시를 지원하지 않으면 빨간 경고 문구로 대체.

### 6.1 대시보드 (`src/pages/StudentDashboard.tsx`, `/student`)

| 포인트 랭킹 | 2×2 | 전체 학생 포인트 순위 Top 10, 본인 행은 테두리 강조 |

### 6.2 내 수업 / 연습 목록 (공용 `PracticeList` 컴포넌트)
같은 컴포넌트를 4곳에서 재사용, 리스트만 다름:
- **내 수업**(`/myclass`): 본인이 등록된 반에 배정된 문제 전체.
- **순서도 연습**(`/practice/flowchart`): 발행된 문제 중 카테고리=순서도.
- **파이썬 코딩**(`/practice/general`): 발행된 문제 중 카테고리=파이썬.
- **블럭 코딩**(`/practice/block`): 발행된 문제 중 카테고리=블럭.

UI: 세로 카드 리스트(제목 + 설명 1줄 미리보기 + 우측 화살표), 클릭 시 풀이 화면으로 이동.
문제 없을 때 "아직 문제가 없습니다" 안내.

### 6.3 문제 풀이 화면 (`src/pages/Solve.tsx`, `/solve/:problemId`)
전체 화면, 좌우 2분할:
- **좌측**: 순서도 문제면 읽기 전용 순서도 캔버스 + 하단에 문제 설명 + 채점 결과.
  일반 문제면 설명 텍스트 + 하단 채점 결과.
  채점 결과는 "N/M 통과 · 점수/만점" + 테스트별 통과 아이콘 목록.
- **우측**: `EditorPanel`(공용, 7장 아래) — 실행 버튼/중단 버튼, 코드 에디터, stdin 입력 +
  출력 콘솔, 우측 하단 "제출" 버튼(제출 시 모든 테스트를 백그라운드에서 돌려 자동 채점).
- 코드 임시저장(500ms 디바운스, 브라우저 로컬), 선생님이 "수업하기" 화면에서 보고 있을 경우
  실시간으로 코드가 브로드캐스트됨(학생 쪽엔 아무 표시 없음, 백그라운드 동작).

### 6.4 포인트 상점 — 학생 (`src/components/student/ShopPanel.tsx`, `/student/shop`)
- 상단: 제목 + 보유 포인트(P) 표시.
- **상품 카드 그리드**(2~5열): 썸네일 + 이름 + "가격P · 재고 N" + 상태별 버튼 텍스트/비활성:
  "구매 요청" (기본) / "요청 중"(이미 대기중 요청 있음) / "품절"(재고 0) / "포인트 부족".
- **구매 내역 섹션**(카드 그리드 하단): 본인이 요청한 전체 내역, 카드형(썸네일 + 상품명 + 상태
  라벨: 대기중/구매완료(초록)/거절됨(빨강)).

## 7. 순서도 편집기 (공용, `FlowchartCanvas` + `FlowNode`)
문제 에디터(순서도 카테고리)와 학생 풀이 화면(읽기 전용), 라이브 모니터링에서 재사용.

- **팔레트**(편집 가능 모드에서만): 시작/입력/출력/처리/조건/for/while/함수/호출/끝 — 10종 노드.
  드래그로 캔버스에 배치.
- **노드 도형**: 시작·끝 = 알약형(둥근 사각), 조건·while = 마름모, 입력·출력 = 평행사변형,
  for = 육각형(또는 자식을 감싸는 큰 컨테이너 박스), 함수 = 이중 테두리 사각형, 호출 = 좌우
  세로선이 있는 사각형, 나머지 = 각진 사각형. 더블클릭으로 라벨 인라인 편집.
  기본 배색은 흰 배경 + 짙은 남색(#111827) 테두리/텍스트, 노드별 커스텀 색상 지정 가능.
- **연결**: 네 방향(상하좌우) 핸들에서 자유롭게 드래그해 화살표 연결(마커 화살표 끝).
- **DSL 가져오기**: 텍스트 기반 미니 언어(들여쓰기로 블록 표현: start/end, input/output/process,
  if/elif/else, for/while, def)를 붙여넣으면 캔버스를 자동 레이아웃으로 대체.
- **자동 정렬** 버튼(레이아웃 아이콘) 존재.
- 읽기 전용 모드(학생 풀이 화면, 라이브 모니터링)에서는 팔레트/편집 UI 없이 캔버스만 표시.

## 8. 공용 코드 에디터 패널 (`EditorPanel`)
문제 에디터 + 풀이 화면에서 공용:

  호출이 있는데 stdin이 비어 있으면 실행 전에 경고 토스트.

## 10. 공통 상태 패턴
- **로딩**: 대부분 "불러오는 중…" 회색 텍스트, 전용 스피너 없음.
- **빈 상태**: "아직 ~가 없습니다." 류의 안내 텍스트, 빈 상태 일러스트 없음.
- **에러/성공 피드백**: 화면 상단 중앙 토스트(`sonner`)만 사용, 인라인 폼 에러 메시지 없음.
- **삭제 확인**: 전부 브라우저 기본 `confirm()` 사용(커스텀 확인 모달 아님) — 디자인 개선 시
  검토 포인트가 될 수 있음.
- **실시간 요소**: 접속 중인 학생(presence), 라이브 코드 브로드캐스트, 문제 목록(Realtime 구독)
  은 별도 새로고침 없이 자동 갱신됨.
